lightnode REST API:

Во всех методах есть параметр connectionId. Его создает метод connect для каждого клиента

GET /ping
	пинг
	req:
		connectionId 
			- идентификатор соединения
				если null, то нода просто отвечает pong
		 		если указан, то ping обновляет таймер соединения. иначе disconnect (на данный момент не проверяется)

GET /connect 
	подключиться к ноде
	req:
		apiVersion
			- версия API. сейчас 'v1' (на данный момент не проверяется)
		apiKey
			- ключ API (на данный момент не проверяется)
	res:
		connectionId
			- нода создает новое соединение и отвечает json с идентификатором соединения по которому 
				следует отсылать остальные запросы

GET /disconnect
	отключиться от ноды
	req:
		connectionId
			- идентификатор соединения

GET /nodeTickers
	получить список курсов
	req:
		connectionId
			- идентификатор соединения
	res:
		[] массив с тикетами в формате fetch

GET /nodeState
	получить состояние ноды
	req:
		connectionId
			- идентификатор соединения
	res:
		объект с состоянем ноды
		{
           	id,
            startTime,
            uptime,
            lastUpdate
        }
GET /poolNotifications
	получить нотификации
	req:
		index
			- если -1, то получить последние сообщения (по-дефолту)
			- если >= 0, то получить нотификации со смещением (для просмотра истории)
		count
			- количество возвращаемых нотификаций

GET /nodeOnOff
	управление нодой
	req:
		onoff
			'on' - включить ноду (фитчинг)
			'off' - выключить ноду (фитчинг)
			'shutdown' - завершить процесс (для теста)
